Structuration - Naissance d'un protocole

Structuration d'un protocole  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
La structuration renonce à une conception monolithique pour laquelle toutes les fonctions sont intégrées dans un unique code informatique, au profit d'une approche modulaire où les fonctions sont regroupées par affinités.



Chaque module contribue à exécuter un ensemble de fonctions cohérentes sur des informations spécifiques.

Les principes étant énoncés, prenons un peu de temps pour agencer les différents composants afin que ce modèle puisse être interopérable et intégrer ultérieurement de nouvelles fonctionnalités.



L'interopérabilité est essentielle dans le domaine des réseaux et plus généralement de l'informatique. Une technologie devient rapidement obsolète si elle ne peut s'intégrer et communiquer avec d'autres.
