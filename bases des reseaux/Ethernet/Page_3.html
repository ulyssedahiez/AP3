Expérience  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Cet exemple présente une trame Ethernet capturée sur un réseau local avec le logiciel Wireshark.



Cette trame est représentée en mode " raw" , où chaque octet est représenté en hexadécimal et où ils sont regroupés en ligne de 16 octets. La position relative de la ligne dans la trame est indiquée sur la gauche.

L'analyse d'une trame serait fastidieuse dans ces conditions. Le logiciel identifie chaque champ de la trame et représente l'ensemble sous forme détaillée comme ce qui est présenté dans les paragraphes ci-dessous.

Vous pouvez réaliser vous-même l'analyse de cette trame en téléchargeant le logiciel Wireshark et en ouvrant le fichier qui contient la capture des trames.

Avec Wireshark, vous disposez d'une lecture globale présentant l'encapsulation des différents protocoles.



Tout d'abord l'entête Ethernet avec les adresses source et destination, puis le message. Dans le cas présent, le message est une requête ARP ( Address Resolution Protocol).
