Gestion des collisions - Protocole Ethernet

Padding  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Le " padding" est utilisé pour compléter la trame Ethernet et obtenir un total minimal de 64 octets, permettant ainsi de détecter les collisions de trames en toutes circonstances.

Cette notion dépasse le cadre de ce cours mais peut s'avérer nécessaire pour identifier les champs d'une trame.

La présentation brute de la trame met en évidence une trame suivie d'une suite d'octets à 0 pour le "padding", ce qui correspond à la réalité du point de vue position.




L'analyse détaillée de la trame met en évidence l'encapsulation d'un message ARP dans une trame Ethernet et la suite d'octets à 0 du "padding" est positionnée dans la trame Ethernet. Cette représentation "logique" évite de considérer le "padding" comme faisant partie du message ARP.

Commutation et full duplex  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
L'acheminement des trames Ethernet nécessite de les présenter à tous les postes afin que chacun puisse prendre la décision de les accepter ou non.

Sans modifier ce mode de fonctionnement CSMA/CD intrinsèque au protocole, la commutation permet d'orienter chaque trame là où se trouve le destinataire et de réduire le trafic inutile.

Compréhension :   0   0   0   0   0   0     QCM :   0   0   0

L'augmentation du trafic est limitée par l'accroissement du nombre de collisions et les retransmissions qui en découlent.

La commutation associée au mode full duplex permet de supprimer les collisions et d'optimiser le trafic.
