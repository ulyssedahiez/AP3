Définitions - Protocole Ethernet

Définitions  
Adresses

Les adresses Ethernet sont composées de 6 octets.

Code du protocole

Quelques exemples :
0x806 pour ARP
0x800 pour IP.

Couches physique et liaison

Elles font référence aux couches n°1 et 2 du modèle OSI.

Voir chapitre "encapsulation".

Ethernet II

La version II du protocole Ethernet a été développée pour intégrer davantage de protocoles dans sa trame.

L'encapsulation

Les trames sont imbriquées les unes dans les autres comme des poupées russes, jusqu’à contenir un éventuel message.

Simple

Transmettre des données en quantité limitée, d’une application à une autre, en contrôlant l’intégrité par un checksum.

Payload

Un payload est une suite d’octets correspondant à un message ou un protocole encapsulé dans la trame d’un autre protocole.

Couches

Les protocoles sont structurés en différents modules pour être indépendants les uns envers les autres.

Les messages générés par une couche supérieure sont encapsulés par une couche inférieure lors de l’émission. A l’inverse lors de la réception.

Prise de parole

A quel moment peut-on envoyer un message sans perturber l’acheminement des autres ?

Le mode CSMA/CD, Carrier Sense Multiple Access with Collision Detection (accès multiple avec écoute de porteuse et détection de collision), utilisé par le protocole Ethernet est un principe basé sur la politesse lors de la prise de parole.

D'intégrer

L’intégration de messages se réalise par l’encapsulation : un entête renseigne sur ce qu’il faut réaliser avec un contenu. C’est un principe communément utilisé par tous les protocoles.

Couche physique

La nomenclature des couches est décrite par le modèle OSI, Open Systems Interconnection.

couche liaison

La nomenclature des couches est décrite par le modèle OSI, Open Systems Interconnection.

ARP

Ce protocole permettra d’avoir connaissance de l’adresse Ethernet du destinataire.

L'identification

L’identification est réalisée par les adresses Ethernet.

Suite alternée de 1 et de 0

10101010 ...

Deux bits à 1

... 10101011

Checksum

Le checksum utilise un algorithme CRC32.

Little-endian ou big-endian

Représentation d’une donnée de plusieurs octets

Raw

Les octets sont représentés en séquence sans regroupement par fonctions.

Padding

Le padding est utilisé pour compléter une trame Ethernet avec une suite de 0.

Commutation

Par autoapprentissage, les équipements réseaux connaissent la localisation de chaque poste.

Full duplex

Half ou full duplex correspond à un canal de communication unidirectionnel en alternat ou bidirectionnel simultané.

Préambule

Le préambule ne contient pas d’informations utiles relatives à l’acheminement d’un message, il assure une fonction "technique" de synchronisation.

64 octets minimum

Par convention, une trame Ethernet doit comporter au minimum 64 octets pour permettre de détecter les collisions de trames en toutes circonstances sur une distance maximale séparant les postes.

IEEE 802.3

Ethernet 802.3 est la première version du protocole Ethernet qui nécessite d’intégrer une trame 802.2 pour identifier les applications. Aujourd’hui, il est réservé à certains usages particuliers.

Collisions de trames

Le protocole Ethernet se base sur le mode CSMA/CD, Carrier Sense Multiple Access with Collision Detection (accès multiple avec écoute de porteuse et détection de collision), pour détecter une collision et réémettre une trame.

RFC

Les RFC (Request For Comments) sont des documents de référence qui décrivent, spécifient, aident à l’implémentation, standardisent et débattent de la majorité des normes, standards, technologies et protocoles dont la Communauté Internet a besoin.
