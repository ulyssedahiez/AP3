Trame - Protocole Ethernet

Préambule  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Le préambule tient un rôle spécifique en précédant la trame Ethernet. Il est élaboré par la couche physique du protocole Ethernet.



Etant composé d'une suite alternée de 1 et de 0, le préambule permet à la réception de la trame de synchroniser l'horloge sur celle de l'émetteur afin d'éviter les erreurs de phase lors de l'échantillonnage des bits composant la trame.

Ces premiers octets peuvent être partiellement perdus lors de cette synchronisation le temps de caler la phase, mais cela n'a aucun impact puisqu'ils sont prévus à cet effet.

Cette suite se termine par deux bits à 1 consécutifs pour marquer le début des informations utiles relatives à la trame Ethernet. Après échantillonnage, celles-ci pourront être restituées à la couche liaison.

Ces notions ne rentrent pas dans le cadre de ce cours mais peuvent s'avérer nécessaires pour aborder la lecture d'un article décrivant les champs d'une trame Ethernet.

Checksum  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Dans l'exemple précédent, la trame Ethernet capturée comportait un entête, un payload et des octets de "padding", aucun autre élément n'y figurait.



Cependant, la trame Ethernet se termine par un checksum ( CRC ) de 4 octets que Wireshark ne présente pas.

Le checksum permet de contrôler la parité de la trame à la réception, et ainsi de corriger 1 bit en erreur ou d'en détecter davantage. Ces 4 octets font partie de la trame Ethernet et rentrent dans le décompte des 64 octets minimum à transmettre (préambule exclu).



En aucun cas, le protocole Ethernet n'est capable de demander à l'expéditeur la retransmission d'une trame incomplète ou identifiée comme erronée au niveau de sa parité. Cela fait partie de la simplicité de sa conception.

Traduire des informations binaires sur un support.
Il est indispensable de convertir les 0 et 1 en signaux compatibles avec le support de transmission (câble cuivre, fibre optique ou radio fréquence).
Identifier chaque poste.
Les adresses Ethernet source et destination permettent d’identifier les postes concernés.
Identifier une erreur de transmission.
Le checksum (CRC ou FCS) est chargé de contrôler l’intégrité de la trame reçue.
Demander la retransmission d'une trame erronée.
Une trame erronée peut être corrigée à la réception ou ignorée si le nombre d’erreurs est trop important. Le protocole Ethernet ne prend en compte qu’une partie de l’intégrité de la transmission.
