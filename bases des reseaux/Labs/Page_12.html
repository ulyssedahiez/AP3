Lab routage local - Manipulation de protocoles

Prérequis  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Cours Ethernet et IP
Préambule  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Etant donné qu'une machine peut disposer de plusieurs interfaces réseau, les trames doivent être envoyées par celle qui correspond à la destination. Il est donc essentiel de faire le lien entre l'adresse IP de destination et l'interface réseau appropriée.
Objectif  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Ce lab illustrera le comportement du protocole IP dans l'utilisation d'une table de routage où les entrées correspondent à différents cas de figure.
Configuration  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Connectez d'une part, deux machines virtuelles Tiny Core Linux VM1 et VM2 sur un même switch virtuel isolé (VMware vmnet2).

Connectez d'autre part, l'une des deux précédentes machines virtuelles VM1 et une troisième Tiny Core Linux VM3 sur un même autre switch virtuel isolé (VMware vmnet3).


Dans un premier temps, initialisez l'adresse IP de l'interface réseau "eth0" de chaque machine connectée sur vmnet2 en sélectionnant leur valeur dans une même classe IP 1 (ifconfig nom_interface adresse_IP).

Attention, vous n'initialiserez pas les autres interfaces réseau "eth0" ou "eth1" des machines connectées sur vmnet3.


Identifiez ensuite les adresses Ethernet et IP de l'interface réseau "eth0" ou "eth1" de chaque machine VM1, VM2 et VM3 (ifconfig nom_interface).

Relevez le contenu des tables de routage de chaque machine VM1, VM2 et VM3 (route -n).


Puis, configurez les différentes machines virtuelles :
- VM1 : (dispose de deux interfaces) en mode console
- VM2 et VM3 : en écoute des trames présentées (tcpdump -i nom_interface)

Lab 1  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement du protocole IP dans l'utilisation d'une table de routage où les entrées correspondent aux paramètres des interfaces réseau comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Envoyez avec la commande "ping" une trame ICMP request depuis la VM1 avec pour adresse de destination, l'adresse IP unicast correspondant à celle de la VM2 (ping adresse_IP).

Points d'attention :
- Contenu de la table de routage de la VM1
Lab 2  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement du protocole IP dans le cas où deux interfaces réseau sont dans la même classe IP comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Initialisez l'adresse IP de l'interface réseau "eth0" de chaque machine connectée sur vmnet3 en sélectionnant leur valeur dans la même classe IP 1 que les autres interfaces réseau connectées à vmnet2, mais différentes des autres adresses IP en terme de valeur.

Envoyez avec la commande "ping" une trame ICMP request depuis la VM1 avec pour adresse de destination, l'adresse IP unicast correspondant à celle de la VM2 puis de la VM3 (ping adresse_IP).

Points d'attention :
- Contenu de la table de routage de la VM1
- Trames reçues
Lab 3  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement du protocole IP dans le cas où une interface réseau n'a pas de correspondance avec la table de routage comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Supprimez l'entrée de la table de routage de la VM1 correspondant au réseau de la VM2 (route del -net classe_IP netmask masque_réseau).

Envoyez avec la commande "ping" une trame ICMP request depuis la VM1 avec pour adresse de destination, l'adresse IP unicast correspondant à celle de la VM2 (ping adresse_IP).
Lab 4  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement du protocole IP dans le cas où une entrée de la table de routage adresse spécifiquement une seule machine située dans un autre réseau Ethernet comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Ajoutez une nouvelle ligne dans la table de routage de la VM1 afin que la VM2 soit accessible sans interférence avec le réseau de la VM3 (route add -net|-host classe_IP netmask masque_réseau dev nom_interface)

Envoyez avec la commande "ping" une trame ICMP request depuis la VM1 avec pour adresse de destination, l'adresse IP unicast correspondant à celle de la VM2 et de la VM3 (ping adresse_IP).

Points d'attention :
- Contenu de la table de routage de la VM1
- Trames reçues
