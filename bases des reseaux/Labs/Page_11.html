Lab acceptation d'une trame Ethernet - Manipulation de protocoles

Prérequis  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Cours Ethernet et IP
Préambule  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
En suivant les principes de fonctionnement du protocole Ethernet, une trame est désencapsulée si l'adresse de destination correspond à l'adresse Ethernet de l'interface de réception.

L'ensemble des traitements réalisés par le protocole Ethernet est centralisé sur la carte réseau. Ils sont donc spécifiques à chaque interface réseau.
Objectif  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Ce lab illustrera le comportement d'une interface réseau dans le traitement d'un datagramme IP unicast encapsulé dans une trame Ethernet II dont l'adresse de destination correspond à l'interface de réception ou à celle d'une autre interface que celle ayant réceptionné la trame.
Configuration  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Connectez deux machines virtuelles Tiny Core Linux VM1 et VM2 sur un même switch virtuel isolé (VMware vmnet2).

Ajoutez une carte réseau à la machine virtuelle VM2 et connectez cette interface sur un autre switch virtuel isolé (VMware vmnet3).

Prenez la précaution de vérifier que l'interface réseau "eth0" de la machine VM2 est connectée sur vmnet2 (et non l'interface eth1). Pour cela, VMware vous permet d'identifier l'adresse Ethernet de chaque carte réseau (advanced ...) et ainsi d'identifier dans linux le nom de l'interface correspondante.


Initialisez l'adresse IP de l'interface réseau "eth0" de chaque machine en sélectionnant leur valeur dans la même classe IP (ifconfig nom_interface adresse_IP). Ces interfaces doivent être celles connectées au switch virtuel vmnet2.

Attention, n'initialisez pas l'autre interface réseau "eth1" de la VM2 connectée sur vmnet3, mais activez la simplement (ifconfig nom_interface up).


Dans un premier temps, identifiez l'adresse Ethernet de l'interface réseau "eth0" ou "eth1" de chaque machine VM1 et VM2 (ifconfig nom_interface).


Vous utiliserez un outil d'analyse réseau plus approprié sur la VM2 pour capturer les trames reçues (tcpdump -i nom_interface). Le script python "dump" n'est en effet pas adapté pour capturer les trames adressées à une autre interface que celle de réception.


Puis, configurez les différentes machines virtuelles :
- VM1 : en mode console
- VM2 : (dispose de deux interfaces) en écoute des trames présentées (script tcpdump)

Travail préparatoire  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Vous trouverez ci-dessous les indications pour compléter les documents demandés dans le travail préparatoire des labs suivants :
- L'identification des machines (nom) et leur interface (adresse Ethernet)
- La contribution de chaque couche dans l'élaboration de la trame (flèches des couches vers les champs)
- La trame émise (valeur des champs)
- Le parcours et le sens de propagation de la trame (flèches)
- Le comportement de chaque couche dans le processus de réception et de traitement de la trame (flèches des champs vers les couches, éventuellement vers la corbeille)
Lab 1  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement d'une interface réseau dans le traitement d'un datagramme IP unicast encapsulé dans une trame Ethernet II comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Générez avec scapy une trame ICMP ping request.

Cette trame sera encapsulée dans un datagramme IP avec pour adresse de source l'adresse IP correspondant à celle de la VM1, et pour adresse de destination l'adresse IP correspondant à celle de la VM2.

Ce datagramme IP sera encapsulé dans une trame Ethernet II avec pour adresse de source l'adresse Ethernet correspondant à celle de la VM1, et pour adresse Ethernet de destination l'adresse celle de l'interface "eth0" de la VM2.

Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement du protocole Ethernet dans le traitement d'une trame unicast dont l'adresse de destination correspond à celle d'une autre interface que celle ayant réceptionné la trame comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Dans les mêmes conditions, générez avec scapy une trame ICMP ping request, excepté que pour l'adresse Ethernet de destination vous indiquerez celle de l'interface "eth1" de la VM2.
