Configuration réseau - Manipulation de protocoles

Présentation générale  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Quatre configurations de base sont disponibles pour interconnecter une interface réseau d'une machine virtuelle à son environnement (le host qui héberge les machines virtuelles, les connexions locales et internet via le filaire ou le wifi, les autres machines virtuelles) :
- Bridge - VMnet0 (extérieure au host, accès lan ou internet)
- Host only - VMnet1 (connexion directe avec le host, pas d'internet)
- NAT - VMnet8 (connexion directe avec le host, accès lan ou internet)
- Custom - VMnetxx (machine virtuelle isolée du host et d'internet), plusieurs switches virtuels sont proposés

Le choix s'établit en fonction du besoin de communication entre la machine virtuelle et son environnement. Les différents switches virtuels sont disponibles dès le lancement de VMware.

Une machine virtuelle peut disposer de plusieurs interfaces réseau, chacune pourra être paramétrée selon les besoins. Des interfaces réseau peuvent être ajoutées ou supprimées via le menu de configuration. Vous pouvez identifier l'interface réseau par son adresse Ethernet accessible depuis le menu de configuration et l'adresse peut être modifiée si nécessaire.

VMware identifie toutes les interfaces disponibles que le host détient et propose d'en retenir quelques unes.
