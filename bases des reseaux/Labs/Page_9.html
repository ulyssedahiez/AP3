Lab correspondance adresses Ethernet et IP, ARP - Manipulation de protocoles

Prérequis  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Cours Ethernet, IP et ARP
Préambule  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
L'utilisation de deux référentiels d'adressage impose de respecter dans le cadre d'un réseau local la correspondance entre les adresses Ethernet et IP.

L'adressage Ethernet joue un rôle "local" de remise du payload, l'adressage IP d'identification du destinataire du message.
Objectif  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Ce lab illustrera le comportement d'une interface réseau dans le traitement d'un datagramme IP encapsulant de simples données ou une trame ICMP.
Configuration  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Connectez deux machines virtuelles Tiny Core Linux sur un même switch virtuel isolé (VMware vmnet2).


Dans un premier temps, initialisez l'adresse IP de l'interface réseau "eth0" de chaque machine en sélectionnant leur valeur dans la même classe IP (ifconfig nom_interface adresse_IP).


Ensuite, identifiez les adresses Ethernet et IP de l'interface réseau "eth0" de chaque machine VM1 et VM2 (ifconfig nom_interface).


Puis, démarrez les scripts sur les différentes machines virtuelles :
- VM1 : sous l'environnement scapy pour injecter des trames (script scapy)
- VM2 : en écoute des trames présentées (script dump)

Travail préparatoire  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Vous trouverez ci-dessous les indications pour compléter les documents demandés dans le travail préparatoire des labs suivants :
- L'identification des machines (nom) et leur interface (adresse)
- La contribution de chaque couche dans l'élaboration de la trame (valeur des champs et flèches)
- La trame émise (valeur des champs)
- Le parcours et le sens de propagation de la trame (flèches)
- Le comportement de chaque couche dans le processus de réception et de traitement de la trame (valeur des champs et flèches)
Configuration préalable  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Avant de réaliser les labs suivants, vous viderez le cache ARP de la VM2 en désactivant et en réactivant son interface réseau "eth0" (ifconfig nom_interface down, ifconfig nom_interface up. Vous pourrez ensuite vérifier le contenu du cache ARP (arp -an).

Vous pourrez utiliser un second terminal (ctrl alt f2) pour ne pas quitter l'environnement scapy.

De même, vous démarrerez sur la VM1 dans un second terminal un outil d'analyse réseau plus approprié afin de capturer toutes les trames (tcpdump -i nom_interface). Le script python "dump" n'est en effet pas suffisamment rapide pour capturer toutes les trames si elles se succèdent.
Lab 1  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement d'une interface réseau dans le traitement d'un datagramme IP encapsulé dans une trame Ethernet II, dont les adresses IP de source et de destination sont correctement renseignées, et le payload de simples données comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Générez avec scapy un datagramme IP avec pour adresse de source l'adresse IP correspondant à celle de la VM1, et pour adresse de destination l'adresse IP correspondant à celle de la VM2.

Le champ "protocole" du protocole IP aura pour valeur 0xFE correspondant à un usage d'expérimentation. Ainsi il vous sera possible d'y ajouter un payload composé du message "hello" sans faire appel à d'autres protocoles.

Ce datagramme IP sera encapsulé dans une trame Ethernet II pour laquelle vous définirez les valeurs des champs d'adresses et ether type en conformité avec l'objectif d'adresser la VM2.
Lab 2  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement d'une interface réseau dans le traitement d'un datagramme IP encapsulé dans une trame Ethernet II, dont le payload est une trame ICMP comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Dans les mêmes conditions, générez avec scapy une trame ICMP ping request encapsulée dans le datagramme IP précédent.

N'oubliez pas de mettre à jour le datagramme IP afin d'encapsuler la trame ICMP.

Points d'attention :
- Echange de trames supplémentaires
- Contenu du cache ARP des VM1 et VM2
Lab 3  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Imaginez un scénario permettant depuis la VM1 d'envoyer une requête ICMP vers la VM2, et de déclencher une réponse ICMP adressée à la VM3.

Mettez ce scénario en oeuvre et vérifiez en le fonctionnement avec l'outil d'analyse réseau tcpdump, option -e pour en obtenir le détail au niveau de la couche liaison (tcpdump -i nom_interface -e).
