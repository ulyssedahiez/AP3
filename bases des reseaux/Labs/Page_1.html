VMware - Manipulation de protocoles

Installation de VMware Workstation Player ou VMware Fusion  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Pour réaliser les labs, vous aurez besoin de l'hyperviseur VMware pour disposer d'un environnement de simulation utilisant des machines virtuelles linux.

Téléchargez VMware Workstation Player pour Windows ou Linux

Téléchargez VMware Fusion pour MacOS
Prérequis  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Une machine virtuelle préconfigurée basée sur Tiny Core Linux est mise à votre disposition sous la forme d'un fichier .ovf.

Vous la trouverez dans le répertoire des labs sous forme de trois fichiers :
- Tiny_core_linux_python_scapy.ovf ( description de la configuration de la machine virtuelle)
- Tiny_core_linux_python_scapy-disk1.vmdk ( machine virtuelle)
- Tiny_core_linux_python_scapy.mf ( signature des fichiers .ovf et .vmdk)

Pour sauvegarder ces fichiers, utilisez le clic droit de la souris et sélectionnez l'option "save link as".
Chargement d'une machine virtuelle  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
L'utilisation de VMware Workstation Player nécessite de lancer autant de fois l'hyperviseur que de machines virtuelles, alors que VMware Workstation Pro permet de les mutualiser dans une seule interface.


Création d'une machine virtuelle :
- Lancez VMware Workstation Player
- Choisissez une machine virtuelle en sélectionnant le fichier ".ovf" (open a virtual machine)
- Donnez-lui un nom afin de la différencier des autres machines virtuelles


Afin d'isoler les machines virtuelles de toute interaction réseau avec la machine hôte, vous configurerez l'interface réseau des machines virtuelles pour qu'elles soient toutes connectées sur un switch isolé, par exemple vmnet2, vmnet3. Vous n'utiliserez pas vmnet0, vmnet1 et vmnet8 car la machine hôte est connectée sur ces switches.

Configuration de l'interface réseau :
- Editez la configuration de la machine virtuelle pour connecter le "Network adapter" sur le switch virtuel souhaité (Edit virtual machine settings)


Lancement de la machine virtuelle :
- Démarrez la machine virtuelle (Play virtual machine)


Si vous obtenez une erreur concernant la virtualisation lors du lancement de VMware, activez la virtualisation dans le BIOS de votre machine hôte.

Touches de fonction

Pour quitter une machine virtuelle et reprendre la main sur la souris : ctrl alt

Pour disposer d'un second terminal : alt f2 (login root)

Pour revenir au premier écran : alt f1

Pour redémarrer une machine virtuelle linux : reboot

Pour arrêter une machine virtuelle linux : poweroff
Préparation des labs  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
1) Pour réaliser les labs, vous aurez besoin dans un premier temps de 3 machines virtuelles. Créez-les dès à présent en les connectant sur le switch virtuel vmnet2.


2) Pour vérifier que toutes les machines virtuelles communiquent correctement, vous pouvez lancer le script "dump" sur l'ensemble des machines virtuelles afin de capturer une trame de test que vous enverrez ensuite.

La commande "./dump" permet de capturer et d'analyser les trames reçues. Pour stopper la capture : ctrl maj c


3) Pour réaliser le test, vous basculerez l'une des machines virtuelles sur un second terminal et vous lancerez le script "test".

La commande "./test" permet d'envoyer une trame de test sur le réseau auquel elle est connectée.


Si toutes les machines sont correctement configurées et connectées sur le même vswitch, la trame envoyée est affichée sur chacune d'elles.
