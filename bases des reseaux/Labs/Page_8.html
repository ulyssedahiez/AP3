Lab encapsulation d'un datagramme IP unicast et broadcast - Manipulation de protocoles

Prérequis  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Cours Ethernet, IP et encapsulation
Préambule  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Les protocoles Ethernet 802.3 et Ethernet II ont la même structure d'adressage, mais se différencient respectivement par le champ "longueur" qui est remplacé par le champ "ether type".

Le protocole Ethernet 802.3 permet de transmettre des messages mais ne dispose pas de champ permettant d'identifier la nature des informations encapsulées ou les applications qui s'y rapportent. C'est la raison pour laquelle les concepteurs ont ajouté ultérieurement un second entête 802.2. Ces ajustements ont complexifié la structure des trames et ajouté de nouvelles contraintes.

Le protocole Ethernet II simplifie l'encapsulation des trames de nombreux protocoles. Le champ "ether type" du protocole Ethernet II permet d'identifier différentes applications ou protocoles de la couche 3 réseau en charge du traitement de son payload.

Le protocole Ethernet 802.3 a permis d'illustrer les fondamentaux de l'encapsulation de simples données issues d'une application située juste au-dessus du protocole Ethernet; le protocole Ethernet II va maintenant le remplacer dans les labs suivants.

Objectif  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Ce lab illustrera le comportement d'une interface réseau dans le traitement d'un datagramme IP unicast ou broadcast encapsulé dans une trame Ethernet II.

Vous vous rendrez compte qu'il faudra disposer de l'adresse Ethernet du destinataire pour mener à bien la transmission des informations.
Configuration  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Connectez trois machines virtuelles Tiny Core Linux sur un même switch virtuel isolé (VMware vmnet2).


Dans un premier temps, initialisez l'adresse IP de l'interface réseau "eth0" de chaque machine VM1, VM2 et VM3 en sélectionnant leur valeur dans la même classe IP (ifconfig nom_interface adresse_IP).


Ensuite, identifiez les adresses Ethernet et IP de l'interface réseau "eth0" de chaque machine VM1, VM2 et VM3 (ifconfig nom_interface).


Puis, démarrez les scripts sur les différentes machines virtuelles :
- VM1 : sous l'environnement scapy pour injecter des trames (script scapy)
- VM2 et VM3 : en écoute des trames présentées (script dump)

Vous trouverez ci-dessous les indications pour compléter les documents demandés dans le travail préparatoire des labs suivants :

Lab 1  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement d'une interface réseau dans le traitement d'un datagramme IP unicast encapsulé dans une trame Ethernet II comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Générez avec scapy un datagramme IP avec pour adresse de source l'adresse IP correspondant à celle de la VM1, et pour adresse de destination l'adresse IP correspondant à celle de la VM2.

Le champ "protocole" du protocole IP aura pour valeur 0xFE correspondant à un usage d'expérimentation. Ainsi il vous sera possible d'y ajouter un payload composé du message "hello" sans faire appel à d'autres protocoles.

Ce datagramme IP sera encapsulé dans une trame Ethernet II pour laquelle vous définirez les valeurs des champs d'adresses et ether type en conformité avec l'objectif d'adresser la VM2.
