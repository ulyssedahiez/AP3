Lab adresses Ethernet unicast et broadcast - Manipulation de protocoles

Prérequis  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Cours Ethernet
Préambule  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Le protocole Ethernet permet de différencier trois types de destinataire :
- Une unique machine
- Un groupe de machines
- L'ensemble des machines d'un réseau local.
Objectif  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Ce lab illustrera le comportement d'une interface réseau dans le traitement d'une adresse Ethernet unicast ou broadcast.

Afin de ne prendre en compte que l'adressage du protocole Ethernet, les trames injectées seront de type Ethernet 802.3.

Configuration  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Connectez trois machines virtuelles Tiny Core Linux sur un même switch virtuel isolé (VMware vmnet2).


Dans un premier temps, identifiez l'adresse Ethernet de l'interface réseau "eth0" de chaque machine VM1, VM2 et VM3 (ifconfig nom_interface).


Puis, démarrez les scripts sur les différentes machines virtuelles :
- VM1 : sous l'environnement scapy pour injecter des trames (script scapy)
- VM2 et VM3 : en écoute des trames présentées (script dump)

Travail préparatoire

Vous trouverez ci-dessous les indications pour compléter les documents demandés dans le travail préparatoire des labs suivants :
- L'identification des machines (nom) et leur interface (adresse Ethernet)
- La contribution de chaque couche dans l'élaboration de la trame (flèches des couches vers les champs)
- La trame émise (valeur des champs)
- Le parcours et le sens de propagation de la trame (flèches)
- Le comportement de chaque couche dans le processus de réception et de traitement de la trame (flèches des champs vers les couches, éventuellement vers la corbeille)
Lab 1  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le comportement d'une interface réseau dans le traitement d'une adresse Ethernet unicast comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Générez avec scapy une trame Ethernet 802.3 avec pour adresse de destination, l'adresse Ethernet unicast correspondant à celle de la VM2.

Concernant l'adresse de source, vous la choisirez de façon judicieuse.

Vous initialiserez le champ "longueur" à 0 afin de forcer scapy à utiliser le mode raw, vous permettant ainsi d'ajouter seulement un payload.

Vous ajouterez un payload composé du message "hello" sans faire appel à d'autres protocoles.
