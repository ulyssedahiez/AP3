Analyse de trames - Manipulation de protocoles

Script dump  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
En complément de ce qu'apporte scapy pour la génération de trames, un script python "dump.py" permet d'identifier et d'analyser les trames reçues par l'interface réseau "eth0" de la machine qui l'exécute. Voici quelques éléments pour en comprendre le fonctionnement :

- L'adresse Ethernet de destination de la trame est toujours présentée.

- Si l'adresse de destination de la trame ne correspond pas à l'adresse Ethernet de l'interface, la trame est résumée en une ligne pour indiquer qu'elle a été reçue mais non acceptée.

- Si l'adresse de destination de la trame correspond à l'adresse Ethernet de l'interface, la trame est détaillée pour indiquer qu'elle a été reçue et acceptée pour être désencapsulée.

En option, pour capturer une ou plusieurs trames reçues sur une interface réseau, il vous faut préalablement disposer de son nom.

L'interface doit être préalablement activée avant de pouvoir être utilisée.

La capture se fait avec la commande sniff() : variable_python = sniff(iface="nom_interface", count=nombre_de_trames_a_capturer)
