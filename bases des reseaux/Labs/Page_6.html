Lab diffusion Ethernet - Manipulation de protocoles

Prérequis

Cours Ethernet
Préambule  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Le protocole Ethernet se compose de deux couches :
- Liaison (couche n°2) : adressage d'une trame
- Physique (couche n°1) : acheminement d'une trame

La notion de "trame" correspond à l'ensemble des informations échangées, c'est à dire les composants d'identification du protocole et les données de l'application.
Objectif  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Ce lab mettra en évidence le principe de diffusion d'une trame Ethernet sur un réseau local. Il illustrera l'une des fonctions du protocole Ethernet, le CSMA/CD (Carrier Sense Multiple Access with Collision Detection / accès multiple avec écoute de porteuse et détection de collision) qui régit le partage du canal de communication au niveau de la couche physique.

Il illustrera les différents comportements d'une interface réseau lors de la réception et du traitement d'une trame Ethernet, et notamment la fonction d'adressage de la couche liaison.

Afin de ne prendre en compte que l'acheminement d'une trame et l'adressage du protocole Ethernet, les trames utilisées seront de type Ethernet 802.3.

Configuration  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Connectez trois machines virtuelles Tiny Core Linux sur un même switch virtuel isolé (VMware vmnet2).


Dans un premier temps, identifiez l'adresse Ethernet de l'interface réseau "eth0" de chaque machine VM1, VM2 et VM3 (ifconfig nom_interface).


Puis, démarrez les scripts sur les différentes machines virtuelles :
- VM1 : sous l'environnement scapy pour injecter des trames (script scapy)
- VM2 et VM3 : en écoute des trames présentées (script dump)

Lab 1  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Préparation
En suivant les indications données dans le paragraphe "travail préparatoire", illustrez sur le document joint le principe de diffusion du protocole Ethernet et d'identification du destinataire comme indiqué dans la partie "réalisation" du présent lab.


Réalisation
Générez avec scapy une trame Ethernet 802.3 avec pour adresse de destination, une adresse Ethernet unicast prise au hasard.

Concernant l'adresse de source, vous la choisirez de façon judicieuse.

Vous initialiserez le champ "longueur" à 0 afin de forcer scapy à utiliser le mode raw, vous permettant ainsi d'ajouter seulement un payload.

Vous ajouterez un payload composé du message "hello" sans faire appel à d'autres protocoles.
